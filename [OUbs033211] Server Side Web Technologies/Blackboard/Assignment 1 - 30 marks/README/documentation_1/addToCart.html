My Cart Functionality - Technical Documentation

1. Overview

The My Cart page allows users to review the items added to their cart, adjust the quantity of each item, remove items, and see real-time updates on the total price of their selected products. Additionally, users can opt to save items for later.

2. Key Features

2.1. Cart Item Display

  • Each item added to the cart is displayed in a structured format that includes:
    • Product Image: A thumbnail of the product.
    • Product Details:
      • Title
      • Description
      • Author (if available)
      • Date of Publication
      • Category
      • Available Quantity (to reflect stock)
      • Price
    • Quantity Selector: Users can increase or decrease the number of units for each item using plus and minus buttons.
    • Save for Later: Allows the user to move the item to a wishlist or another list for future purchase without keeping it in the cart.
    • Remove: Allows the user to delete the item from the cart.

2.2. Quantity Management

  • Description:

    • Each item in the cart has a quantity selector with + and - buttons to increment or decrement the quantity.
    • The system ensures the quantity does not exceed the available stock or fall below one.

  • Real-Time Updates:

    • Every time the quantity is changed, the cart is updated in real-time using AJAX to adjust the total price.

2.3. Total Price Calculation

  • Real-Time Calculation:

    • The Price Details section on the right side of the cart page updates in real-time to reflect:
      • The total price for the items added (based on item price multiplied by quantity).
      • Delivery charges (which may be FREE for certain orders or calculated based on shipping logic).
      • Amount Payable: The final total after adding all the items and delivery charges.

  • Formula:

    • For each item: Total Item Price = Unit Price Ã— Quantity
    • Amount Payable: Sum of all item prices + Delivery Charges

2.4. Cart Actions

  • Save for Later:

    • If the user clicks the "Save for Later" button, the item is moved out of the cart and placed in a separate list, where the user can add it back later.
    • This action does not affect the cart total.

  • Remove:

    • Clicking "Remove" removes the item from the cart and updates the total price in real-time.
    • If the cart becomes empty after removing all items, a message is displayed to inform the user that the cart is empty.

3. Price Details Panel

  • Content:
    • Price: Displays the total cost for the number of items in the cart.
    • Delivery Charges: Shows the delivery cost. It could be a fixed amount, dynamic based on delivery location, or FREE if a threshold (e.g., total cart value) is met.
    • Amount Payable: This is the sum of the product prices plus delivery charges, updated dynamically based on the user's cart changes.

4. Data Flow

4.1. Client-Side (AJAX Updates)

  • Adding/Removing Items:

    • When the user adds or removes an item from the cart, or adjusts the quantity, AJAX requests are sent to the server to update the cart in real-time without reloading the page.

  • Quantity Changes:

    • The plus and minus buttons trigger client-side AJAX requests to:
      • Validate the new quantity against available stock.
      • Recalculate the total price.

  • Price Calculation:

    • AJAX ensures that the Price Details (total price, delivery charges, and amount payable) are recalculated and displayed as the cart content changes.

4.2. Server-Side

  • Cart Persistence:

    • For logged-in users, the cart is stored in the database and linked to their account.
    • For guest users, the cart is stored in the session and may expire when the session ends.

  • Real-Time Price Updates:

    • The server recalculates the total price whenever the cart is updated, ensuring the amount payable is accurate.
    • Delivery charges may also be computed based on the number of items or the total cart value.

5. Error Handling

  • Stock Availability:

    • If the user tries to add more items than are available, the system will display an error message indicating the available quantity.

  • Session Expiry:

    • If the guest session expires, the cart data will be lost, and the user will be prompted to log in or re-add items to the cart.

6. User Experience

6.1. Visual Updates

  • Quantity Management:
    • When the user increases or decreases the quantity of an item, the page reflects the changes instantly through visual updates without needing a page refresh.

6.2. Notifications

  • Item Removal:
    • When an item is removed, a notification may be displayed confirming that the item was successfully removed.

6.3. Empty Cart State

  • If the user removes all items from the cart, the page will show an Empty Cart message with a prompt to continue shopping.
