Search and Filter Functionality - Technical Documentation

1. Overview

The Search and Filter functionality enables users to search for products based on specific criteria such as Product Name, Category, Price Range, and other attributes. This feature also includes pagination to break down the product catalogue into manageable pages, improving user experience and load times. Filters are applied using OR conditions, allowing users to find products that match any of their selected criteria.

2. Key Features

2.1. Search Bar

  • Product Name Search:
    • Users can enter a product name or keyword into the search bar.
    • The system performs a search for products that match the entered text within the product name.

2.2. Filter Options

  • Category Filter:

    • Users can select a category from the dropdown to filter products within that specific category.

  • Price Range Filter:

    • Users can enter a Minimum Price and Maximum Price to filter products within a specified price range.
    • Products are displayed if their price falls between the minimum and maximum values.

2.3. OR Condition Logic

  • The filters (product name, category, price range) are applied using OR conditions, meaning:
    • If the user selects multiple filters, any product matching one or more of the conditions will be displayed.
    • Example: If the user selects "Category: Cloud" and sets a price range of 1000 - 3000, products from the "Cloud" category or products within the price range will be displayed.

2.4. Search Button

  • After entering the search term or selecting filter options, the user clicks the Search button to apply the filters.
  • The search results are displayed based on the selected criteria, updating the product list dynamically.

3. Product Display with Pagination

3.1. Pagination System

  • The product list is divided into pages using pagination to avoid overloading the page with too many results at once.

  • At the bottom of the page, users can navigate between different pages using pagination controls (e.g., 1, 2, Next).

  • Functionality:

    • A set number of products (e.g., 10) are displayed per page.
    • Users can navigate through pages to view more products.
    • The pagination is updated dynamically based on the search and filter results.

3.2. Real-Time Pagination

  • When users apply filters or search for products, the system dynamically updates the pagination:
    • If fewer results are returned, fewer pagination numbers will be displayed.
    • The system ensures that only products matching the search criteria are displayed across multiple pages.

4. Data Flow

4.1. Client-Side (Search and Filters)

  • The search bar and filters trigger client-side requests using AJAX to send the search criteria to the server.
  • After filtering, the client receives the filtered product list and displays it on the page.

4.2. Server-Side (Filtering and Pagination)

  • The server processes the search query and filters applied by the user.
  • The server returns the relevant products based on the applied OR conditions.
  • The server ensures the pagination reflects the number of matching products and splits them across multiple pages as needed.

5. Error Handling

  • Invalid Price Range:

    • If the user enters an invalid price range (e.g., min price > max price), a validation message is displayed, and the search is not performed.

  • No Search Results:

    • If no products match the search criteria, the page will display a message such as "No products found."

  • Pagination Error:

    • If a user navigates to a page that no longer exists (e.g., filtering reduces the number of pages), the pagination will be reset to page 1.

6. User Experience

6.1. Real-Time Feedback

  • Once filters are applied, the product list and pagination update in real-time, providing a seamless browsing experience.
  • The current search term and filter selections remain visible, allowing users to modify them as needed.

6.2. Navigation

  • Users can easily switch between product pages using pagination controls.
  • The Search button triggers immediate filtering, reducing the number of steps to perform the search.

7. Security Considerations

  • SQL Injection Prevention:

    • Input from the search bar and filters is sanitized to prevent SQL injection or any form of malicious data manipulation.

  • Data Integrity:

    • Ensure that the pagination updates correctly based on the search results to maintain consistency in the displayed data.
